# 3D Quotation Ecosystem — Видение продукта

## Миссия

Создать экосистему инструментов, которая позволит команде **мгновенно отвечать клиентам** на вопросы о стоимости 3D-печати и моделирования, **исключить ошибки** в расчётах и **накапливать данные** для принятия бизнес-решений.

---

## Проблемы, которые решаем

| Проблема | Боль | Влияние |
|----------|------|---------|
| Долгий расчёт стоимости | Клиент ждёт КП 1-2 дня | Упущенные заказы |
| Ошибки в ценниках | Пересчёт вручную, разные формулы | Потеря маржи или клиента |
| Нет истории расчётов | Не помним, что считали | Повторная работа |
| Нет аналитики | Не знаем реальную себестоимость | Неоптимальные цены |
| Неудобно на выезде | Нужен компьютер и Excel | Медленная реакция |

---

## Пользователи

### Основные
- **Менеджер по продажам** — быстро отвечает клиентам, формирует КП
- **Инженер-технолог** — точный расчёт с учётом технологии

### Вторичные
- **Руководитель** — аналитика, ценообразование, отчёты
- **Клиент** (в будущем) — самостоятельный предварительный расчёт

---

## Продукты экосистемы

### 1. Подробный калькулятор (Web)
**Для:** Инженер, менеджер за компьютером
**Что делает:** Точный расчёт по STL-файлу для 4 технологий
**Ключевое:** Загрузка STL → автоматический расчёт объёма → цена

### 2. Экспресс-калькулятор печати (Web + TG Mini App)
**Для:** Менеджер на выезде, быстрый ответ клиенту
**Что делает:** Вилка цены по габаритам + сложность за 30 секунд
**Ключевое:** Не нужен файл, работает на телефоне

### 3. Экспресс-калькулятор моделирования (Web + TG Mini App)
**Для:** Менеджер при оценке работ по 3D-моделированию
**Что делает:** Оценка трудозатрат по экспертным критериям
**Ключевое:** Тип изделия + сложность → часы → стоимость

### 4. Аналитика и дашборды (Web)
**Для:** Руководитель, финансист
**Что делает:** Статистика, анализ ценников, ФОТ, расходники
**Ключевое:** Данные для принятия решений

## Принципы разработки

1. **MVP first** — сначала минимум для проверки гипотезы
2. **Итерации** — релиз каждые 1-2 недели
3. **Единый бэкенд** — все продукты используют один API
4. **Mobile-friendly** — работает на телефоне
5. **Данные с первого дня** — всё сохраняется в БД
